## What is Software Architecture?

**Software architecture shows the big picture, how the system is shaped and structured at a high level, without focusing on details.**  
**It defines the main building blocks, how they connect, and the rules that keep everything organized and working together.**


#### Software Design vs Software Architecture vs System Design:


**Software Design** => The plan for building each small part correctly, focusing on internal details like algorithms, classes, and functions.

**Software Architecture** => The big picture of the system and how its main parts connect and work together, focusing on the structure rather than the small stuff.

**System Design** =>  The complete plan for making the system work in the real world, covering databases, servers, networks, data flow, and the strategies needed to scale.



## Architectural Patterns:

Architectural Patterns are standard ways to structure a software system. They show how the main components are organized, connected, and work together.
